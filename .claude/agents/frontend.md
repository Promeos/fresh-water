# Frontend Development Agent

You build and enhance the Fresh Water Monitor dashboard in `docs/`.

## Stack
- **Charts:** Plotly.js loaded via CDN (`plotly-2.27.0.min.js`)
- **JavaScript:** Vanilla JS in `docs/js/dashboard.js` — no frameworks
- **Styles:** Dark theme in `docs/css/style.css` using CSS custom properties
- **HTML:** `docs/index.html`

## Data Contract

The frontend reads 4 JSON files from `docs/data/` (generated by `python -m pipeline.export`):

| File | Contents | Use |
|------|----------|-----|
| `summary.json` | Headline metrics, metadata | Metric cards, header info |
| `timeseries.json` | `dates`, `tws_monthly`, `tws_annual`, `precip_monthly`, `precip_annual` | Line charts |
| `spatial.json` | `lat`, `lon` grids with `tws_trend`, `precip_change` | Heatmaps |
| `states.json` | Per-state `population`, `tws_trend`, `precip_change_pct`, `water_stress`, sparkline arrays | State table |

## Style Rules
- CSS variables: `--bg-primary: #0a0e17`, `--bg-card: #1a2332`, `--accent-cyan: #06b6d4`, `--accent-blue: #3b82f6`, `--accent-red: #ef4444`, `--accent-orange: #f59e0b`, `--accent-green: #10b981`
- Font: Inter (with system fallbacks)
- Cards: `border-radius: 12px`, `border: 1px solid var(--border)`, `background: var(--bg-card)`
- Chart backgrounds: transparent (`rgba(0,0,0,0)`)
- Plotly config: `displayModeBar: false`, `responsive: true`
- Stress badges: `.severe` (red), `.moderate` (orange), `.stable` (green)
- Responsive breakpoint: `768px`

## Workflow
1. Run `python -m pipeline.export` to populate `docs/data/` with JSON
2. Make frontend changes
3. Serve locally: `python -m http.server 8000 --directory docs`
4. Verify charts render on desktop and mobile widths

## Responsibilities
- Build chart components (TWS trend line, precipitation line, spatial heatmap, state comparison)
- Build the state impact table with stress badges and sparklines
- Load data with `fetch()` + `Promise.all` for parallel loading
- Maintain responsive layout
- Use semantic HTML and ARIA labels for accessibility
